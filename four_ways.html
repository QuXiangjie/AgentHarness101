Four ways to write the agent loop

同一个玩具 agent — 一个工具 get_weather，提问 "What's the weather in XAA?" — 用四种方式写。 它们做的事完全一样：调用模型 → 如果它要用工具，就执行并把结果喂回去 → 重复直到它停止。 变的只是 这个循环有多少由你写、有多少被框架藏起来。

← 你写全部框架写全部 →

1 · Raw HTTP

自己 POST JSON、解析字典。循环完全暴露。

2 · Claude SDK

同样的循环，但是有类型对象。SDK 只是传输层。

3 · LangChain

循环消失，进了 create_agent。

4 · LangGraph

循环又出现了，变成一张状态图。

四种方式的差异一览

维度1 · Raw HTTP2 · Claude SDK3 · LangChain4 · LangGraph
谁写循环 你 — 显式 while True 你 — 显式 while True 框架 — 隐藏 你，但写成图（节点 + 边）
"我做完了吗" 判断 resp["stop_reason"] != "tool_use" response.stop_reason != "tool_use" 藏在 .invoke() 里 tools_condition（一条路由边）
工具调度 你 — run_tool() + 手动拼 tool_result 你 — run_tool() + 手动拼 tool_result 自动 — 从 @tool 签名推断 自动 — ToolNode
返回结构 原始 dict（resp["content"]） 类型对象（response.content、block.type） result["messages"] 里的消息对象 result["messages"] 里的消息对象
工具定义 手写 JSON schema 手写 JSON schema @tool + 类型注解 + docstring @tool + .bind_tools()
鉴权 / 重试 / 类型 你 SDK 负责 框架负责 框架负责
依赖 requests anthropic langchain + langchain-anthropic langgraph + langchain-anthropic
harness 代码量 ~30 行（最多） ~30 行 ~8 行（最少） ~15 行（声明式）
适合场景 理解 harness 到底是什么 生产级 agent，要完全掌控循环 快速交付一个标准 agent 分支 / 多 agent / human-in-loop
核心就一句话：四种方式里模型完全相同 — 变的只有 harness。 #1 和 #2 的循环是一段你能从头读到尾的 while；#3 里它消失了；#4 里它又以图的形式回来。 亲手把 #1 写一遍，另外三个就不再是魔法 —— 你会认出同一个循环藏在每一个里面。

核心循环并排对比 高亮 = 循环逻辑

1 · rawharness_text.py — Raw HTTP 你掌控循环deps: requests
# 自己 POST JSON 并解析字典
messages = [{"role": "user", "content": "What's the weather in XAA?"}]
while True:                                       # <-- harness 循环
    resp = requests.post(API_URL, headers=HEADERS, json={...}).json()
    messages.append({"role": "assistant", "content": resp["content"]})
    if resp["stop_reason"] != "tool_use":        # 模型完成了
        break
    tool_results = []                              # 执行每个被请求的工具
    for block in resp["content"]:
        if block["type"] == "tool_use":
            tool_results.append({"type":"tool_result", "tool_use_id":block["id"],
                                 "content": run_tool(block["name"], block["input"])})
    messages.append({"role": "user", "content": tool_results})
2 · rawharness_sdk.py — Claude SDK 你掌控循环deps: anthropic
# 完全相同的循环 —— 只是用类型对象代替字典
client = anthropic.Anthropic()
while True:                                       # <-- 仍然是你的循环
    response = client.messages.create(model=..., tools=TOOLS, messages=messages)
    messages.append({"role":"assistant", "content": response.content})
    if response.stop_reason != "tool_use":        # 类型属性，不是字典 key
        break
    for block in response.content:               # block.type / block.id / block.input
        if block.type == "tool_use":
            run_tool(block.name, block.input)      # ...同样地 append tool_result
3 · rawharness_langchain.py — LangChain 框架掌控循环deps: langchain, langchain-anthropic
from langchain.agents import create_agent

@tool
def get_weather(city: str) -> str:
    """Get the current weather for a city."""
    return _get_weather(city)

agent = create_agent(model="anthropic:claude-opus-4-7", tools=[get_weather])
result = agent.invoke({"messages": [{"role":"user","content":"What's the weather in XAA?"}]})
print(result["messages"][-1].content)
# 没有 while、没有 stop_reason、没有工具调度 —— .invoke() 跑完整个循环
4 · rawharness_langgraph.py — LangGraph 循环 = 你声明的一张图deps: langgraph, langchain-anthropic
llm = ChatAnthropic(model="claude-opus-4-7").bind_tools(tools)
def call_model(state): return {"messages": [llm.invoke(state["messages"])]}

graph = StateGraph(MessagesState)
graph.add_node("model", call_model)
graph.add_node("tools", ToolNode(tools))
graph.add_edge(START, "model")
graph.add_conditional_edges("model", tools_condition)  # = "if stop_reason != tool_use" 的判断
graph.add_edge("tools", "model")                       # <-- 回环的那条边
app = graph.compile()
result = app.invoke({"messages": [{"role":"user","content":"What's the weather in XAA?"}]})
四种方式共享的部分：工具本身只在 tools/weather.py 里写一次 （get_weather() + JSON schema），被每个 harness 导入 —— 所以变的只是循环风格，工具从不重复。
